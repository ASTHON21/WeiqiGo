# Identity & ID Logic Architecture (Technical Manifest)

This document outlines the identification systems and ID relationships within the Weiqi Go platform for AI agent context and architectural understanding.

## 1. Core Identity Layer (Authentication)
- **Mechanism**: Firebase Anonymous Authentication.
- **Identifier**: `UID` (28-character alphanumeric string).
- **Scope**: Persistent per browser instance (via Firebase Auth's internal IndexedDB persistence).
- **Security Role**: Every request to Firestore is backed by `request.auth.uid`. No unauthenticated writes are permitted.

## 2. Entity ID Relationships

### A. UserProfile Entity
- **Path**: `/userProfiles/{userId}`
- **ID Logic**: The `{userId}` MUST exactly match the Firebase Auth `UID`.
- **Primary Key**: `id` (String) - Mirrors the Document ID.
- **Reference For**: `playerBlackId`, `playerWhiteId`, `winnerId`.

### B. Game Entity
- **Path**: `/games/{gameId}`
- **ID Logic**: Auto-generated by Firestore (`addDoc`).
- **Composite State**:
    - `playerBlackId`: UID of the black player.
    - `playerWhiteId`: UID of the white player.
- **Room Logic**: The `{gameId}` serves as the unique Room/Routing ID (`/game/online/{gameId}`).

### C. Move Entity
- **Path**: `/games/{gameId}/moves/{moveId}`
- **ID Logic**: Sub-collection pattern. `{moveId}` is auto-generated.
- **Ordering**: Sorted by `moveNumber` (Integer) and `timestamp` (ServerTimestamp) for replay reconstruction.

## 3. Local Persistence Logic (Local-First)
- **Storage Key**: `goMasterHistory` (localStorage).
- **ID Format**: `practice-{timestamp}`.
- **Purpose**: Decoupled from Cloud IDs. Used for offline practice records and SGF exports.
- **Data Shape**: `GameHistoryEntry` interface.

## 4. AI Defense Logic via IDs
- **Verification**: Firestore Security Rules compare `request.auth.uid` against `resource.data.playerBlackId` or `resource.data.playerWhiteId`.
- **Write Restriction**: Only an ID-verified participant can push to the `moves` sub-collection.
- **Integrity**: IDs are immutable once a game status is set to `finished`.

## 5. System Constants
- **Board Sizes**: 9, 13, 19 (Integer).
- **Color IDs**: `black`, `white` (String).
- **Rules IDs**: `chinese`, `territory` (String).

---
*Note: FingerprintJS and Custom Device Header IDs have been deprecated and removed in favor of Firebase Auth UID for superior security and AI-tamper resistance.*
